# 管理者マニュアル — shadowing-web（モデル版）

## 管理画面へのアクセス

```
https://shadowing-web.onrender.com/admin
```

ログインパスワードは `ADMIN_PASSWORD` 環境変数に設定した値です。

---

## ライセンスキーの発行

### 管理画面から発行

1. 管理画面上部の「キー生成」セクションを開く
2. 以下を設定して「生成」ボタンをクリック:

| 項目 | 説明 | 例 |
|------|------|-----|
| 商品 | 発行するライセンス種別 | `base` |
| 枚数 | 発行する枚数（1〜1000） | `1` |
| 最大台数 | 1キーあたりの使用可能台数（1〜10） | `5` |
| メモ | 注文番号や購入者名など | `Booth注文#001` |

### ライセンス種別一覧

| 種別 | 内容 | 用途 |
|------|------|------|
| `base` | ベースアプリ（デフォルト100題） | 基本購入者全員 |
| `pack:hotel` | ホテルパック単品 | パック単品購入者 |
| `pack:toeic` | TOEICパック単品 | パック単品購入者 |
| `pack:travel_vol1` | 旅行vol1単品 | パック単品購入者 |
| `pack:travel_vol2` | 旅行vol2単品 | パック単品購入者 |
| `pack:jiji_vol1` | 自字vol1単品 | パック単品購入者 |
| `pack:jiji_vol2` | 自字vol2単品 | パック単品購入者 |
| `bundle:travel_set` | base + 旅行vol1 + vol2（3キーセット） | 旅行セット購入者 |
| `bundle:toeic_set` | base + TOEIC（2キーセット） | TOEICセット購入者 |
| `bundle:hotel_set` | base + ホテル（2キーセット） | ホテルセット購入者 |
| `bundle:jiji_set` | base + 自字vol1 + vol2（3キーセット） | 自字セット購入者 |
| `bundle:all` | base + 全パック（7キーセット） | 全部入り購入者 |

> **バンドルについて:** `bundle:travel_set` を発行すると base・travel_vol1・travel_vol2 の3枚のキーが一括生成されます。購入者には3枚全て送付してください。

## 販売フロー

### Boothなどでの販売手順

1. **注文受付** → 注文番号を控える
2. **キー発行** → 管理画面で購入商品に対応したキーを生成（メモ欄に注文番号を記載）
3. **キー送付** → `docs/email-*.txt` のテンプレートを使ってメールで送付
4. **確認** → ライセンス一覧でアクティベーション状況を確認

### メールテンプレートの場所

```
docs/
├── email-base.txt        ベースアプリ用
├── email-hotel.txt       ホテルパック用
├── email-toeic.txt       TOEICパック用
├── email-travel.txt      旅行パック用
└── email-jiji.txt        自字パック用
```

## ライセンスの管理

### ライセンス一覧の見方

管理画面の一覧に表示される情報:

| 列 | 説明 |
|----|------|
| ハッシュ（先頭8文字） | ライセンスの識別ID |
| キー | 発行したライセンスキー（SHAD-...） |
| 商品 | ライセンス種別 |
| 使用台数 | アクティベーション済み台数 / 上限 |
| 作成日時 | キー発行日 |
| メモ | 発行時に入力したメモ |

### デバイス認証の取消

#### 全デバイス取消（キーリセット）

転売・紛失などでキーを無効化したい場合:
1. 対象ライセンスの「デバイス一覧」を開く
2.「全て取消」ボタンをクリック
3. 全デバイスの認証が無効になり、キーが再利用可能になる

#### 特定デバイスのみ取消

1. デバイス一覧から対象デバイスを見つける
2. そのデバイスの「取消」ボタンをクリック

### ライセンスの削除

不要なライセンスを完全に削除する場合は「削除」ボタンをクリック。**この操作は取り消せません。**

## バックアップ・復元

### バックアップ

管理画面 → 「バックアップ」ボタン → `licenses-backup-YYYY-MM-DD.json` がダウンロードされます。

**推奨:** 週1回以上バックアップを取得してください。Render.com のディスクは永続ですが、万一のサービス障害に備えてローカルに保存してください。

### 復元

1. 管理画面 → 「バックアップから復元」
2. ダウンロードしたJSONファイルをアップロード
3. 既存データとマージされます（重複キーはスキップ、新規キーのみ追加）

## サーバー管理（Render.com）

### 環境変数の変更

1. Render.com ダッシュボード → `shadowing-web` → **Environment**
2. 変更したい変数を編集
3. **Save Changes** → 自動再デプロイ

| 変数 | 変更が必要な場面 |
|------|----------------|
| `OPENAI_API_KEY` | APIキーを新しいものに変更するとき |
| `ADMIN_PASSWORD` | パスワードを変更するとき |

### 再デプロイ

GitHub の `main` ブランチにプッシュすると自動的に再デプロイされます。

手動でデプロイしたい場合: Render.com → **Manual Deploy** → **Deploy latest commit**

### ログの確認

Render.com → **Logs** タブでサーバーログをリアルタイム確認できます。

TTS エラーの場合は以下のようなログが出ます:
```
[tts] You exceeded your current quota...
```
→ OpenAI のクレジット残高を確認してください: https://platform.openai.com/billing

### ディスク使用量の確認

TTS キャッシュは `/data/tts-cache/` に蓄積されます（上限400MB）。

Render.com → **Metrics** → **Disk Usage** で確認できます。

## CLIツール（Render Shell）

Render.com の Shell タブから直接コマンドを実行できます。

```bash
# ライセンスキーを1枚発行
node tools/gen-license.mjs base 1

# バンドルキーを発行（メモ付き）
node tools/gen-license.mjs bundle:travel_set 1 "Booth注文#001"

# ライセンス一覧表示
node tools/admin.mjs list

# 特定キーの無効化
node tools/admin.mjs revoke SHAD-XXXX-XXXX-XXXX
```

## TTSコスト管理

### 料金目安

OpenAI TTS の料金: **$0.015 / 1,000文字**（約¥2〜3 / テキスト1件）

### コスト削減の仕組み

1. **ディスクキャッシュ**: 一度生成した音声は `/data/tts-cache/` に永続保存。同じテキストを別ユーザーが再生しても課金なし
2. **ブラウザキャッシュ**: 同一セッション中の再再生は完全無料

### APIキーの使用量確認

https://platform.openai.com/usage で月次使用量を確認できます。

## トラブルシューティング

| 症状 | 原因 | 対応 |
|------|------|------|
| 音声が出ない | OpenAI クレジット残高不足 | https://platform.openai.com/billing でチャージ |
| 音声が出ない | OPENAI_API_KEY 未設定 | Render.com 環境変数を確認 |
| ライセンス認証できない | licenses.json の読込エラー | Render.com ログを確認・再起動 |
| 管理画面にログインできない | ADMIN_PASSWORD 未設定 | Render.com 環境変数を確認 |
| パックが表示されない | data/packs/ に未コピー | サーバーを再起動（自動コピーが走る） |

## セキュリティ注意事項

- `ADMIN_PASSWORD` は推測されにくい文字列を設定してください
- ライセンスキーは購入者以外に公開しないでください
- `licenses.json` のバックアップファイルには全ライセンス情報が含まれます。取り扱いに注意してください
- GitHub リポジトリに `.env` や `data/` がプッシュされていないことを定期的に確認してください（`.gitignore` で除外済み）
